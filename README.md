Project Overview of Merge Sort VS Heap Sort

In this project, we compare the performance of three sorting algorithms, Merge Sort, Heap
Sort, and a variation called Double Pop Heap Sort. All
three algorithms have the same theoretical time complexity
of O(n log n), which suggests they should perform similarly
as input size increases. However, algorithms with the same
time complexity can behave differently due to factors such
as implementation details, constant factors, and the types of
operations they perform. The objective of this project is to
analyze how these algorithms perform as input size increases
and to determine which one is more efficient in terms of
runtime. So, to ensure a fair comparison, each algorithm is
tested using the same input data across multiple trials, and the
average runtime is recorded. This allows for a more accurate
evaluation of performance. By comparing these algorithms,
this project aims to highlight the difference between theoretical
efficiency and real-world performance and to provide a better
understanding of how algorithm design affects execution time.
